# ClonePlayer Mod — PROJECT_HANDOFF

## 1. Что это за проект

Это админский мод для **Minecraft Forge 1.12.2**, добавляющий управляемых клонов игроков.

Клоны создаются по нику игрока, могут копировать скин, имя, инвентарь и экипировку, а затем управляться командами и через специальный предмет — **Control Stick / Контрольная палочка**.

Проект развивается поэтапно.  
Важно сохранять стабильность рабочей версии и не добавлять крупные системы без согласования.

## 4. Краткая концепция мода

Мод позволяет:
- создать клона игрока по нику
- дать клону поведение:
  - стоять
  - следовать
  - охранять
  - охотиться
  - патрулировать
  - кемперить
  - берсерк
- управлять клонами через команды
- задавать точки через Control Stick
- использовать инвентарь клона
- менять сложность клона
- задавать игнор определённых игроков
- сохранять клонов после перезахода

## 5. Что уже реализовано / рабочая база

### 5.1. Создание клона
Команда:
- `/cloneplayer <ник>`

Поведение:
- создаёт клона как моб-сущность
- если игрок онлайн:
  - копируется имя
  - копируется скин (или как минимум есть логика подгрузки скина)
  - копируется инвентарь
  - копируется броня
  - копируется offhand
- если игрок оффлайн:
  - клон создаётся без вещей
  - имя/skin source сохраняются по нику

---

### 5.2. Команды управления клоном
Реализованы команды вида:
- `/clonecon <клон|id> hunt <игрок>`
- `/clonecon <клон|id> follow <игрок>`
- `/clonecon <клон|id> guard <игрок>`
- `/clonecon <клон|id> stay`
- `/clonecon <клон|id> patrol [игрок1 игрок2 ...]`
- `/clonecon <клон|id> camp`
- `/clonecon <клон|id> berserk`
- `/clonecon <клон|id> difficulty <easy|normal|hard>`

Также:
- `/clonelist`
- `/cloneremove <клон|id>`
- `/cloneremoveall`
- `/clonetp <клон|id>`
- `/clonetphere <клон|id>`
- `/cloneinv <клон|id>`
- `/clonerename <клон|id> <новое имя>`
- `/cloneignore <клон|id|all> <игрок>`
- `/cloneconfig list`
- `/cloneconfig <ключ> <значение>`

---

### 5.3. Режимы поведения
Сейчас концептуально присутствуют режимы:

#### STAY
- стоит на месте
- не двигается
- не преследует цель

#### FOLLOW
- идёт за указанным игроком
- без боевой охраны

#### GUARD
- следует за указанным игроком
- атакует того, кто нанесёт этому игроку урон

#### HUNT
- преследует указанного игрока
- пытается убить его

#### PATROL
- ходит между точками A и B
- точки задаются через Control Stick
- может иметь список “тревожных” игроков
- если один из таких игроков попадает в поле зрения, клон срывается в атаку

#### CAMP
- приходит в заданную точку
- стоит в ней
- атакует цели в радиусе, если они попадают в поле зрения

#### BERSERK
- атакует всё живое, что видит
- игроки, мобы, мирные, нейтральные, враждебные
- кроме ограничений, предусмотренных текущей логикой и ignore

## 6. Control Stick

### Предмет:
- `Control Stick / Контрольная палочка`

### Текущее поведение:
- ПКМ по клону → выбрать клона
- затем ПКМ по блоку:
  - для `PATROL`: ставятся точка A и точка B
  - для `CAMP`: ставится точка лагеря

### Важно:
Control Stick сейчас работает без сложного GUI-мультивыбора.  
GUI-версия Control Stick **обсуждалась**, но на момент этого handoff может быть ещё не реализована.

## 7. Сложность клона

Присутствуют уровни:
- `EASY`
- `NORMAL`
- `HARD`

Сложность влияет на:
- скорость движения
- кулдауны атаки
- кулдаун стрельбы
- разброс стрел

> Точные числа смотреть в `CloneDifficulty.java`.

## 9. Anti-cheese / ломание блоков

Есть логика anti-cheese:
- если цель забирается наверх и клон не может достать
- клон может ломать блоки
- есть временная неуязвимость на время добычи
- после сброса цели даётся краткая “фора”
- затем преследование продолжается

Также обсуждалось дальнейшее развитие:
- клон должен уметь докапываться до игрока под землю
- выбираться из коробки
- использовать инструменты для блоков
- абсолютная охота по координатам

На момент handoff это может быть **частично реализовано или только запланировано** — проверять по коду.

## 12. Сохранение данных

Клоны должны:
- сохраняться после перезахода
- читать и писать NBT

Сохраняются как минимум:
- имя клона
- skin source
- mode
- difficulty
- controlled player
- patrol точки
- camp точка
- ignore-листы
- инвентарь
- часть боевого состояния

Проверять сериализацию в:
- `EntityClonePlayer.java`
- `CloneWorldData.java`

---

## 13. Игнор игроков

Есть система игнора:
- персональный ignore у конкретного клона
- глобальный ignore через world data

Если игрок добавлен в ignore:
- клон не должен атаковать его
- если уже гнался за ним, должен прекращать это поведение
